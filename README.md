# 极客大学「算法训练营-第15期」作业提交仓库


## 仓库目录结构说明

1. `week01/` 代表第一周作业提交目录，以此类推。
2. 请在对应周的目录下新建或修改自己的代码作业。
2. 每周均有一个 `REDAME.md` 文档，你可以将自己当周的学习心得以及做题过程中的思考记录在该文档中。

## 作业提交规则
 
1. 先将本仓库 Fork 到自己 GitHub 账号下。
2. 将 Fork 后的仓库 Clone 到本地，然后在本地仓库中对应周的目录下新建或修改自己的代码作业，当周的学习总结写在对应周的README.md文件里。
3. 在本地仓库完成作业后，push 到自己的 GitHub 远程仓库。
4. 最后将远程仓库中当周的作业链接，按格式贴到班级仓库对应学习周的issue下面。
5. 提交issue请务必按照规定格式进行提交，否则作业统计工具将抓取不到你的作业提交记录。 

详细的作业提交流程可以查阅：https://shimo.im/docs/m5rtM8K8rNsjw5jk/ 


## 注意事项

 如果对 Git 和 GitHub 不太了解，请参考 [Git 官方文档](https://git-scm.com/book/zh/v2) 或者极客时间的[《玩转 Git 三剑客》](https://time.geekbang.org/course/intro/145)视频课程。


学习心得：
由于第一周课程在体验课里面大部分看过，所以就针对第三和第四课进行多次观看。以及前面预习的内容也是反复看了几次，第一周带给我最大的感受就是老师强调的刻意联系，职业训练方法。个人总结一个核心的理论就是简单的问题重复做，反复做，直到信手拈来，融合成自己的条件反射，虽然这个过程是及其痛苦的，并且容易‘走火入魔’而坚持不下去。但是这一周自己咬牙每晚在加班回来之后强迫自己必须完成当天任务以及e刻意反复练习之前做过的简单算法题，我明显感觉到自己对简单程度的一些问题第一反应就是暴力，双指针。这两个很实用的解题方式，这是在没参加算法训练营之前是想不到的（可能由于没人引路不会总结导致即使之前用过这两个方法但是后面碰到类似题目关联不起来），因为刻意，默写题目，感觉就像条件反射似的看道题目就会优先想起这两个方法。 但是偶尔也会翻车，比如超哥给的“位运算”题目。。。。
